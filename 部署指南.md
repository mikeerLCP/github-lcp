# 把 Python + PostgreSQL 项目免费部署到公网 — 通用指南

适用场景：Flask 项目 + PostgreSQL 数据库，3-4 人用，免费、无需自己买服务器。

---

## 用到的平台

| 平台 | 作用 | 费用 | 网址 |
|------|------|------|------|
| GitHub | 存放代码 | 免费 | https://github.com |
| Neon | PostgreSQL 云数据库 | 免费 500MB | https://neon.tech |
| Render | 运行 Flask 服务 | 免费 750h/月 | https://render.com |

## 第一步：准备好代码结构

项目需要包含这些文件：

```
项目根目录/
├── run.py              # 入口：app = create_app()
├── config.py           # 配置：DATABASE_URL 从环境变量读取
├── requirements.txt    # 依赖：flask, gunicorn, psycopg2-binary ...
├── Procfile            # 内容：web: gunicorn run:app -b 0.0.0.0:$PORT
├── .gitignore          # 忽略 __pycache__/ *.pyc .env
├── app/                # Flask 应用代码
└── static/             # HTML/JS/CSS 文件
```

### config.py 关键写法

```python
import os

class Config:
    DB_URL = os.environ.get("DATABASE_URL", "postgresql://localhost/mydb")

    @classmethod
    def get_connection(cls):
        import psycopg2
        return psycopg2.connect(cls.DB_URL)
```

### run.py 关键写法

```python
from app import create_app
app = create_app()  # 模块级 app，gunicorn 需要这个
```

### Procfile 内容

```
web: gunicorn run:app -b 0.0.0.0:$PORT
```

注意 `$PORT` 不要加反斜杠，Render 会自动设置这个环境变量。

## 第三步：创建 Neon 数据库

1. 打开 https://neon.tech → 用 GitHub 登录
2. 点 **Create project**
3. 数据库名填你的项目名
4. 区域选 **Singapore (ap-southeast-1)**（离国内最近）
5. 创建后页面会显示一个 **连接字符串（Connection string）**，长这样：

   ```
   postgresql://用户名:密码@xxx.ap-southeast-1.aws.neon.tech/数据库名?sslmode=require
   ```

   把这个字符串复制下来，后面要用。

6. 点左侧 **SQL Editor** → 粘贴你的建表 SQL → 点 Run

---

## 第四步：部署到 Render

1. 打开 https://render.com → 用 GitHub 登录
2. 右上角 **New +** → **Web Service**
3. 找到你的 GitHub 仓库 → 点 **Connect**
4. 填写配置：

   | 字段 | 值 |
   |------|-----|
   | Name | 任意名称 |
   | Runtime | **Python 3**（不要选 Docker） |
   | Build Command | `pip install -r requirements.txt` |
   | Start Command | `gunicorn run:app -b 0.0.0.0:$PORT` |

5. 点 **Advanced** 展开 → **Environment Variables** → 添加：

   | Key | Value |
   |-----|-------|
   | `DATABASE_URL` | 第三步复制的 Neon 连接串 |

6. 点 **Create Web Service**，等 3-5 分钟部署完成

7. 部署完成后，点左侧 **Environment** → 再检查一遍 `DATABASE_URL` 是否已经设置

---

## 第五步：使用

- Render 会给一个免费域名，类似 `xxx.onrender.com`，这就是你的公网地址
- 发给其他人，浏览器直接访问即可
- 所有人都能看到同一个网站，数据都在 Neon 数据库里共享

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 部署后页面显示 Not Found | gunicorn 没启动 | 检查 Procfile 里 `$PORT` 有没有误写成 `\$PORT` |
| Build 失败 | 缺系统库 | 去掉 pdfplumber、pillow 等需要系统库的依赖，改成函数内按需导入 |
| 首次打开很慢 | 免费版休眠 | 15 分钟不访问自动休眠，第一次打开等 10-20 秒唤醒 |
| 国内访问慢 | 服务器在海外 | 正常现象，可以接受；受不了就买国内云服务器 |
| Render 日志报 `psycopg2` 错误 | 数据库连不上 | 检查 DATABASE_URL 环境变量是否正确 |
